# 使用第三方库

从python / r / JavaScript等“脚本语言”入手学习编程的人可能会对“编译”这个概念感到陌生。

“传统”的编程语言，像 C / C++ / Java等，源代码是源代码，而这些语言的源代码是无法直接被执行的；想要执行，需要先使用相应的“编译器”，将源代码编译成为二进制格式，也就是计算机CPU可以识别的无数010010101格式，然后才可以被运行；对于复杂的C++程序，编译动辄耗时若干小时是很常见的事情：
https://xkcd.com/303/

像Python / R这样而源代码直接就可以被“运行”的编程语言，则被称为“脚本语言” scripting language。
（有时也叫“解释型语言”：程序可以一边输入，一边被解释；不用等待完整输入完毕才能被编译、执行。）

使用“脚本”写出来的程序往往相对简单，可能一个程序往往也就几百，最多几千行；而“编译型语言”写的程序几百万甚至几千万行或更多都有可能。

python这脚本语言在被运行时，其实也是会需要被编译，当我们在命令行console中执行：
`python main.py`
的时候，实际上是在调用`python`这个程序，去编译`main.py`这个源代码文件，然后自动输出结果；如果`main.py`中有引用import到自己写的类库，我们还可以发现在类库目录中自动出现了一个`__pycache__`目录，而目录中有一些`.pyc`格式即python compiled的文件，这些文件便是被编译为的二进制格式的python代码。

因为python的代码往往较少，所以其编译、执行过程相对会比较快，使得用户可能完全没意识到编译过程的存在，而想当然的以为python或者R的代码就是可以被直接执行的“程序”。

这其实也是这些语言作者有意识的行为，他们想要让用这些语言写成的各种类库，在默认提供给别人使用的时候，必须直接提供明文的源代码，而不是编译过后无法被人类阅读的二进制文件；这样使用类库的人，可以直接翻阅类库的实现源码，甚至进行修改，这会有利于整个语言的发展。

这是影响人类文明的：开源精神。

所以，我们会看到python / R里面有无数无所不能的类库，无论我们需要什么，都可以去搜一下是不是网上已经有别人实现了相应的功能，我们直接pip install然后import 一下就可以使用；python类库的丰富程度在各种编程语言中尤为出名：

https://xkcd.com/353/

pip的介绍：https://en.wikipedia.org/wiki/Pip_(package_manager)

除了使用pip安装现成被已经被“打包”好的类库外，我们还能够直接将网上的python代码直接复制到本地来执行。

https://github.com/Wuvist/sgx3000/blob/main/pysgx/dm_test.py

关于python本地类库module的目录结构使用习惯，可以参考：https://docs.python.org/3/tutorial/modules.html
